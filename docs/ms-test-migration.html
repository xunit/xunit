Migrating from MSTest to xUnit.net

In order to assist in migrating unit tests from [MSTest V2](https://github.com/microsoft/testfx) to xUnit.net, it can often be helpful to have unit tests from MSTest and xUnit.net side by side. Tests can then be migrated gradually without breaking the compiler nor the test results in most cases.

  • Step 1: Add nuget package
  • Step 2: Apply changes
  • Step 3: Switch to Xunit
  • Step 4: Remove nuget package

Step 1: Add nuget package

Install xUnit.net nuget packages and MSTest backwards compatibility nuget package [xunit.MSTest](https://www.nuget.org/packages/xunit.MSTest) instead of MSTest nuget packages.

Step 2: Apply changes

Apply all suggested changes from compiler warnings, e.g.

{% highlight diff %} using Microsoft.VisualStudio.TestTools.UnitTesting; -// Warning CS0618 'TestClassAttribute' is obsolete: 'Remove this attribute.' -[TestClass] public class UnitTest1 { - // Warning CS0618 'TestMethodAttribute' is obsolete: 'Use Fact instead.' - [TestMethod] + [Fact] public void TestMethod1() { - // Warning CS0618 'Assert.IsTrue(bool, string)' is obsolete: 'Use True instead.' - Assert.IsTrue(true); + Assert.True(true); } } {% endhighlight %}

Some of these text replacements can be automated with the command line tool from [XUnitConverter](https://github.com/dotnet/codeformatter/tree/master/src/XUnitConverter).
